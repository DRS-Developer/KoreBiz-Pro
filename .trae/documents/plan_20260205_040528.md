O plano a seguir detalha a implementação de um ecossistema de distribuição e instalação para o sistema ArsInstalações, cobrindo automação de deploy, instalador web, compatibilidade com hospedagem compartilhada e documentação.

### **Fase 1: Preparação da Infraestrutura e Configuração Dinâmica**
Para permitir que o mesmo código ("build") rode em diferentes ambientes (Vercel, Hostinger, Local) sem recompilação, precisamos externalizar as configurações.

1.  **Refatoração do Cliente Supabase (`src/lib/supabase.ts`)**:
    *   Alterar a inicialização para buscar credenciais em ordem de prioridade: `window.ENV` (Runtime) > `import.meta.env` (Build time).
    *   Isso permite injetar configurações via arquivo `config.json` em hospedagens compartilhadas.

2.  **Loader de Configuração (`Runtime Config`)**:
    *   Criar script no `index.html` que tenta carregar `/config.json` antes do React iniciar.
    *   Se o arquivo não existir ou as variáveis de ambiente estiverem ausentes, o sistema entra em **Modo de Instalação**.

3.  **Suporte a Hospedagem Compartilhada (Apache/Nginx)**:
    *   Criar arquivo `.htaccess` otimizado para React Router (redirecionando todas as requisições para `index.html`).
    *   Criar script opcional `install.php` para servidores PHP que permite salvar o `config.json` via interface web (contornando a limitação de escrita do navegador).

### **Fase 2: Desenvolvimento do Instalador Web ("Wizard")**
Uma interface amigável estilo "WordPress" que guia o usuário na configuração inicial.

*   **Rota**: `/install` (Acessível apenas se o sistema não estiver configurado).
*   **Etapas do Wizard**:
    1.  **Boas-vindas e Verificação**: Checagem de requisitos (HTTPS, acesso à internet).
    2.  **Conexão com Banco**: Formulário para inserir `Supabase URL` e `Anon Key`. Teste imediato de conexão.
    3.  **Configuração do Banco (Schema)**:
        *   Como navegadores não podem criar tabelas diretamente (segurança), o Wizard exibirá o SQL de migração consolidado.
        *   Botão "Copiar SQL" com instruções claras para colar no Editor SQL do Supabase.
        *   Botão "Verificar Tabelas": O Wizard tenta consultar as tabelas para confirmar se o SQL foi rodado.
    4.  **Criação do Admin**: Formulário para criar o primeiro usuário administrador (via Supabase Auth API).
    5.  **Dados Iniciais (Seed)**: Botão para popular o banco com dados padrão (Configurações do Site, Textos iniciais) usando a API de dados (INSERTs funcionam direto do navegador).
    6.  **Finalização**:
        *   **Vercel**: Exibe as chaves para colar nas Variáveis de Ambiente.
        *   **Hospedagem**: Gera o arquivo `config.json` para download ou salva automaticamente (se `install.php` estiver presente).

### **Fase 3: Automação de Deploy (Vercel & CI/CD)**
1.  **Vercel Deploy Button**: Adicionar link "Deploy to Vercel" no README que clona o repo e pede as variáveis necessárias.
2.  **Otimização do Build**: Garantir que scripts de pré-build (`generate-sitemap`, `generate-static-db`) rodem automaticamente na Vercel.

### **Fase 4: Análise Comparativa e Documentação**
Produção de relatório e manuais técnicos.

1.  **Relatório Comparativo (Markdown)**:
    *   **Vercel**: Gratuito (hobby), alta performance (Edge Network), deploy git-based.
    *   **Hospedagem Compartilhada (Hostinger/KingHost)**: Custo fixo baixo, performance média, requer upload manual (FTP) ou CI/CD customizado, bom para clientes que já têm plano.
    *   **VPS**: Controle total, maior custo de manutenção, escalabilidade vertical.
2.  **Documentação de Instalação (`INSTALL.md`)**:
    *   Passo a passo com screenshots do Wizard.
    *   Guia de Troubleshooting (erros de CORS, RLS policies).
